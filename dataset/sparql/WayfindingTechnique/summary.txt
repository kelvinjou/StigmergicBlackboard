## Core Definition of `WayfindingTechnique`

**`WayfindingTechnique`** is defined as a subclass of `InteractionTechnique`:

```turtle
:WayfindingTechnique rdf:type owl:Class ;
    rdfs:label "Wayfinding Technique"@en ;
    rdfs:subClassOf :InteractionTechnique ;
    rdfs:comment "Methods that assist the user in maintaining and recovering spatial orientation during navigation."@en .
```

### Key Characteristics
- **Fundamental Purpose**: Supports users in developing and using *cognitive maps* of virtual environments
- **Distinguishing Feature**: Unlike `TravelTechnique` (which controls *how* to move), wayfinding techniques address *where* to go and *understanding* spatial relationships
- **Core Challenge**: Minimizing the gap between egocentric (viewer-centered) and allocentric (world-centered) reference frames

## Detailed Analysis of Each Subclass Community

### 1. **TrailCues** & **Breadcrumbs** *(Path-Based Techniques)*

Both rely on **episodic memory of traversed routes** but differ in implementation:

```
TrailCues ──► Continuous record ──► Supports fine-grained backtracking
Breadcrumbs ──► Discrete waypoints ──► Efficient origin-destination links
```

**Relations to other ontology communities:**
- **Supports tasks**: `WayfindingTask` (explicitly stated via `:supportsTask`)
- **Avoids human factor**: Reduces `CognitiveLoad` by externalizing route memory
- **Design principle application**: `SupportSpatialUnderstanding`
